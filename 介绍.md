# DTC 商品创意故事版 Skill 中文介绍

## 1. 一句话定位

这个 skill 把一个真实 SKU 变成可继续做视频 demo 的 DTC 商品创意故事版：先锁商品真相，再由 `DTC Creative Director` 选择创意类型、场景、运动、人物和镜头，形成 `DTC Creative Reference Pack`，最后由 `Storyboard Design Expert` 输出横屏 5 格图片和英文视频脚本。

它不是普通文案工具，也不是直接生一张好看的图。

## 2. 适用输入

- Amazon 商品链接 / ASIN
- Shopify 商品链接
- TikTok Shop 商品链接
- 商品图片
- 商品截图
- SKU brief
- 商品短描述
- 用户上传的参考广告、截图、品牌素材或脚本

输入越接近真实商品证据，输出越稳定。

## 3. 默认输出

默认最终回复只输出三段：

```markdown
## 1. 产品图

## 2. 故事版

## 3. 15 秒或者 30 秒视频脚本文字
```

`## 2. 故事版` 必须是真实图片：一张横向单行 5 格 contact sheet，每格 16:9 横屏。

## 4. 两个内部角色

### DTC Creative Director

不负责出图。它负责在出图前做创意判断。

必须选择：

```text
creative_type
scene_ref
motion_chain
model_ref
camera_plan
script_pattern
picture_fragment_logic
```

它的唯一交付物是 `DTC Creative Reference Pack`。

### Storyboard Design Expert

不重新发明创意方向。它只基于 `DTC Creative Reference Pack` 做两件事：

- 生成横屏 5 格 16:9 故事版图片；
- 写 15 秒或 30 秒英文视频脚本。

## 5. DTC Creative Director 详细说明

DTC Creative Director 的输入是：

```text
Product Truth Card
Ad Reference Decomposition
Category Adapter
```

它按固定顺序执行：

```text
1. Select creative_type
2. Select scene_ref
3. Select motion_chain
4. Select model_ref
5. Select camera_plan
6. Select script_pattern
7. Select picture_fragment_logic
8. Output DTC Creative Reference Pack
```

它借鉴广告结构，不替代商品事实。

## 6. 创意类型库

位置：

```text
references/dtc-creative-pattern-library.md
```

包含：

- problem-solution
- objection-proof
- feature-to-benefit
- mechanism-reveal
- comparison
- routine-integration
- customization-choice
- trust-proof
- sensory-proof
- product-macro
- ugc-demo
- founder-demo

每个 `creative_type` 都有适用场景、买家张力、需要的商品证据、视觉结构、hook 逻辑、proof 逻辑、CTA 逻辑、风险和禁用条件。

## 7. 场景库

位置：

```text
references/dtc-scene-library.md
```

包含：

- studio-tabletop
- bathroom-counter
- kitchen-prep
- desk-setup
- travel-packing
- bedside-routine
- gym-locker
- car-interior
- gifting-table
- unboxing-table
- outdoor-use
- no-human-product-macro

场景必须自然容纳商品，并能支持横屏 16:9 的商品证明。

## 8. 运动库

位置：

```text
references/dtc-motion-library.md
```

每个 motion 都必须有：

```text
start_state -> action -> end_state
```

包含：

- pick-up
- unbox
- align
- apply
- pour
- tap
- plug-in
- wear
- unfold
- fold
- compare
- rotate
- reveal
- hand-pass
- texture-touch
- close-open

运动不是为了好看，是为了让每格能继续作为视频生成关键帧。

## 9. 模特 / 人物库

位置：

```text
references/dtc-model-library.md
```

包含：

- no-human-product-macro
- hand-only
- ugc-creator
- founder
- shopper
- expert
- couple
- parent
- athlete
- office-worker
- home-user

人物不是必须出现。人物出现时必须服务商品证明，不能抢商品。

## 10. 镜头语言库

位置：

```text
references/camera-language-library.md
```

包含：

- locked-off-demo
- top-down-tabletop
- macro-push-in
- handheld-follow
- slow-dolly
- rack-focus
- tabletop-tracking
- orbit-product-reveal
- hero-reveal
- match-cut
- split-screen-comparison
- over-the-shoulder-product-check

每个镜头都有 `video_keyframe_use`。第 3 格通常必须用能证明商品的镜头，比如 `macro-push-in`、`locked-off-demo` 或 `split-screen-comparison`。

## 11. Reference Pack Schema

完整 schema：

```text
references/dtc-director-reference-pack-schema.md
```

简版：

```yaml
dtc_creative_reference_pack:
  product_truth_lock:
  creative_type:
  selected_ad_references:
  scene_selection:
  motion_selection:
  model_selection:
  camera_selection:
  script_pattern:
  picture_fragment_logic:
  rejected_references:
  adaptation_reason:
```

没有这个 pack，不进入故事版图片生成。

## 12. Storyboard Design Expert 规则

Storyboard Design Expert 只能使用 Reference Pack 里的内容：

- product_truth_lock
- creative_type
- selected_ad_references
- scene_selection
- motion_selection
- model_selection
- camera_selection
- script_pattern
- picture_fragment_logic

不能重新选择 creative_type，不能临时换场景、人物、动作或镜头。

## 13. 横屏 5 格图片规则

规则文件：

```text
references/storyboard-video-keyframe-rules.md
```

固定 5 格任务：

```text
1. Hook + Product Entrance
2. Buyer Tension / Context
3. Strongest Product Proof
4. Use Moment / Benefit / Objection Resolution
5. CTA Intent / Purchase Confidence / Hero Frame
```

第 3 格必须是最强商品证明，不能只是普通中间镜头。

## 14. 脚本 Camera / Motion 规则

第三段脚本必须是英文，格式固定：

```markdown
| 时间 | 画面 | Camera / Motion | 屏幕字幕 | 旁白 / 口播 | Reference Logic |
|---|---|---|---|---|---|
```

要求：

- `Camera / Motion` 必须来自 `camera_selection` 和 `motion_selection`。
- `Reference Logic` 必须说明借鉴的结构、场景、动作、镜头或脚本逻辑。
- 屏幕字幕和旁白必须英文。
- 不能新增故事版里没有的新场景、新人物、新卖点或新声明。

## 15. 完成标准

一次完整运行应该得到：

- Product Truth Card 已建立；
- 商品真相优先于所有创意；
- 广告参考或 fallback 参考已记录；
- 每条参考保留 `source_url` 或本地素材路径；
- 每条参考被拆成 `ad_reference`；
- DTC Creative Director 已选择 `creative_type`、`scene_ref`、`motion_chain`、`model_ref`、`camera_plan`；
- 已形成 `DTC Creative Reference Pack`；
- Storyboard Design Expert 只使用 Reference Pack；
- 输出一张横屏 5 格 16:9 故事版图片；
- 每格都是 video-ready keyframe；
- 第 3 格是最强商品证明；
- 第三段英文脚本包含 `Camera / Motion` 和 `Reference Logic`；
- 本地保存 `run.json`、`source_records.json`；
- validator 能通过。
