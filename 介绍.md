# DTC 商品创意故事版 Skill 中文介绍

## 一、定位

这个 skill 的定位是：给一个真实商品，先实时收集和拆解全球 DTC 创意广告参考，再生成可做横屏视频 demo 的商品故事版图片和英文脚本。

它不是普通脚本工具，也不是单纯 AI 生图工具。它服务的是 SKU 级内容生产：

```text
商品链接 / 商品图 / 商品截图 / SKU brief
-> 商品真相锁定
-> 实时广告参考采集
-> 广告结构拆解
-> DTC Creative Reference Pack
-> 横屏 5 格故事版图片
-> 英文视频 demo 脚本
```

最终目标是让一个商品可以更快进入 Shopify、Amazon、TikTok Shop、Meta、YouTube 等平台的短视频创意生产，而不是只得到一段泛泛的广告文案。

## 二、适用输入

支持输入：

- Amazon 商品链接
- Shopify 商品链接
- TikTok Shop 商品链接
- 商品图片
- 商品截图
- ASIN
- SKU brief
- 商品短描述
- 用户上传的参考广告
- 用户给的竞品链接
- 用户给的品牌素材或脚本

输入越接近真实商品证据，输出越稳定。

## 三、默认输出

默认只输出三段：

```markdown
## 1. 产品图

## 2. 故事版

## 3. 15 秒或者 30 秒视频脚本文字
```

`## 2. 故事版` 必须是图片，不是文字分镜。

故事版图片要求：

- 一张横屏 storyboard / contact-sheet 图
- 5 个 panel
- 每个 panel 都是 16:9 横屏构图
- 默认横向单行排列
- 同一商品
- 同一拍摄世界
- 同一光线方向
- 动作连续
- 真实接触阴影
- 商品必须清楚可读

## 四、内部角色

这个 skill 内部有两个角色。

### 1. Live DTC Creative Intelligence Director

负责实时采集和拆解 DTC 广告参考。

它做的事情：

- 分析商品是什么
- 判断商品行为和买家决策问题
- 建立实时采集计划
- 查找公开可访问广告参考
- 使用用户授权账号或用户提供私域内容
- 接收用户上传的广告、截图、脚本或素材
- 保存 `source_url` 或本地素材路径
- 拆解广告结构、场景、人物、动作、镜头、节奏、画面片段、剧本和 CTA
- 形成 `DTC Creative Reference Pack`

### 2. Storyboard Design Expert

负责把商品真相和 Reference Pack 转成故事版图片和视频脚本。

它做的事情：

- 选择内容角度
- 选择 hook
- 选择场景
- 选择模特 / 人物策略
- 选择商品动作和人物动作
- 选择镜头语言
- 生成横屏 5 格故事版图片
- 输出英文视频 demo 脚本

## 五、实时采集来源

采集来源分三类。

### 1. public

公开可访问来源：

- Meta Ad Library
- TikTok Creative Center
- Google Ads Transparency Center
- YouTube
- 品牌官网
- 公开 landing page
- 公开社媒广告页
- 公开广告案例库
- 搜索结果页

### 2. authorized

用户授权或用户账号可访问来源：

- 用户登录状态下的广告库
- 用户自己的广告账号后台
- 用户购买的创意库
- 用户自己的素材库
- 用户提供的私域参考页面
- 用户提供的私密链接

### 3. user_provided

用户直接提供的素材：

- 广告视频
- 广告截图
- 广告链接
- 品牌案例
- 竞品清单
- 已下载素材
- 脚本
- 参考图

## 六、可复制、采集和保存的内容

可以进入 reference pack 的内容：

- 广告原片 / 视频文件
- 视频片段
- 截图 / 缩略图
- 关键画面帧
- 脚本文字
- 口播
- 字幕
- 品牌资产
- landing page 画面
- hook 文字
- CTA 文案
- 评论或社媒上下文
- 产品展示片段
- 模特 / 人物画面
- 动作片段
- 镜头片段
- `source_url`

这些内容用于拆解和生成参考，不是直接把一个广告原样变成最终故事版。

## 七、平台阻挡时的降级策略

如果平台页面打不开、登录失效、广告库阻挡、素材无法下载或无法抓取，按顺序降级：

```text
平台页面
-> 搜索结果摘要
-> 手动链接
-> 用户给参考广告
-> 用户上传素材
-> 内置参考库
```

无论使用哪种方式，都要保留：

```yaml
source_url:
source_type:
local_asset_path:
capture_time:
notes:
```

这样后面可以追溯这个故事版借鉴了哪些广告结构、场景、动作和镜头。

## 八、9 步执行流程

### 1. 证据锁定

先锁定商品真相。

必须确认：

- 商品类型
- 形状 / 轮廓
- 颜色
- 材质 / 纹理
- 包装 / 标签 / logo
- 可见部件
- 尺寸和比例线索
- 商品如何被使用
- 什么结果是可见的
- 哪些声明有证据
- 哪些声明不能写

### 2. 实时广告参考采集

基于商品生成 `Live Crawl Plan`：

```yaml
live_crawl_plan:
  product_category:
  product_behavior:
  buyer_problem:
  main_benefit:
  proof_type:
  competitor_keywords:
  adjacent_category_keywords:
  platform_keywords:
  region_keywords:
  source_priority:
```

采集目标不是找“相同商品”，而是找适合当前 SKU 的 DTC 创意结构。

### 3. 广告拆解

每条参考广告拆成 `ad_reference`：

```yaml
ad_reference:
  source_url:
  source_type:
  platform:
  brand:
  category:
  product_type:
  format:
  aspect_ratio:
  duration:
  original_assets:
    video:
    screenshots:
    keyframes:
    transcript:
    brand_assets:
  hook:
    first_3_seconds:
    hook_type:
    visual_interruption:
  scene:
    location:
    surface:
    lighting:
    props:
  model:
    presence_type:
    role:
    wardrobe:
    continuity_notes:
  motion:
    human_action:
    product_action:
    camera_motion:
    pacing:
  proof:
    proof_mechanic:
    product_detail_shown:
    objection_answered:
  script:
    spoken_line_pattern:
    subtitle_pattern:
    CTA:
  reusable_pattern:
    picture_fragment_logic:
    scene_logic:
    action_logic:
    camera_logic:
    script_logic:
```

### 4. 内容属性卡

内部建立 `Content Attribute Card`：

```yaml
content_attribute_card:
  target_buyer:
  buyer_stage:
  content_angle:
  hook_type:
  core_message:
  product_promise:
  reason_to_believe:
  objection:
  proof_asset:
  cta_intent:
```

内容属性决定故事版，而不是先随便生成好看的画面。

### 5. Reference Pack 选择

把商品真相、内容属性和广告拆解结果转成 `DTC Creative Reference Pack`：

```yaml
dtc_creative_reference_pack:
  product_truth_lock:
  selected_ad_references:
  selected_creative_patterns:
  selected_scene_refs:
  selected_model_refs:
  selected_motion_refs:
  selected_camera_refs:
  selected_script_patterns:
  picture_fragment_refs:
  source_urls:
  rejected_references:
  adaptation_reason:
```

这个 pack 是故事版生成的核心。

### 6. 内容策略选择

确定：

- `shopper_tension`
- `buyer_event`
- `creative_lens`
- `product_entrance`
- `visual_device`
- `scene_anchor`
- `model_strategy`
- `motion_chain`
- `camera_plan`

内容策略必须能拍，能生成图，能继续做视频 demo。

### 7. 横屏故事版图片生成

故事版图片生成指令必须包含：

- 商品真相锁
- 内容属性锁
- 已选广告参考逻辑
- 场景库选择
- 模特 / 人物库选择
- 运动库选择
- 镜头库选择
- 画面片段逻辑
- 剧本结构
- 横屏 16:9 contact-sheet 约束

5 格逻辑：

```text
1. Hook / 商品进入 / SKU 锚点 / 参考 hook 结构
2. 买家疑虑 / 内容角度建立 / 参考场景和人物关系
3. 商品证明 / 机制 / 细节 / 参考 proof mechanic
4. 使用瞬间 / 利益点 / 反对点消除 / 参考动作和镜头运动
5. CTA intent / 购买信心 / 最终商品主导状态 / 参考 CTA 结构
```

### 8. 视频 demo 英文脚本

脚本默认 15 秒。用户要求 30 秒时才写 30 秒。

表格字段：

```markdown
| 时间 | 画面 | Camera / Motion | 屏幕字幕 | 旁白 / 口播 | Reference Logic |
|---|---|---|---|---|---|
```

要求：

- `Camera / Motion` 必须是英文
- `Reference Logic` 必须是英文
- 屏幕字幕必须是英文
- 旁白 / 口播必须是英文
- 不能写故事版里没有的新卖点
- 不能用旁白补救画面没有证明的内容

### 9. 交付前 QA

检查：

- 是否有产品图
- `## 2. 故事版` 是否是真实图片
- 是否横屏
- 是否 5 个 16:9 panel
- 是否同一商品
- 是否同一光线
- 是否动作连贯
- 是否有真实接触阴影
- 是否保存 `source_url` 或本地素材路径
- 是否形成 `DTC Creative Reference Pack`
- 是否完成参考广告拆解
- 是否选择 scene_ref、model_ref、motion_ref、camera_ref
- 脚本是否包含 Camera / Motion 和 Reference Logic
- 商品真相是否高于创意

## 十、商品真相规则

商品最重要。

不能改：

- 商品几何形状
- 商品颜色
- 商品材质
- 商品包装
- logo / 标签
- 尺寸 / 比例
- 使用方式
- 兼容关系
- 功效声明
- 可见结果
- 买家决策逻辑

如果创意和商品真相冲突，删创意，不改商品。

## 十一、真实感规则

故事版图片必须像真实可拍素材。

必须有：

- natural daylight / soft studio daylight
- real shadows
- physical contact shadows
- readable material texture
- realistic lens perspective
- practical tabletop / hand / package / use setup
- product-first composition
- restrained props

避免：

- AI 塑料感
- 悬浮商品
- 不可能阴影
- 过度锐化纹理
- 随机装饰道具
- 幻想光效
- 假 UI
- 假评论卡
- 假平台 badge
- 看不清的小字

## 十二、完成标准

这个 skill 做完一次，应该得到：

- 真实商品证据被锁定
- 实时广告参考或降级参考被收集
- 每条参考保留 `source_url` 或本地素材路径
- 广告参考被拆解成结构、场景、人物、动作、镜头、节奏、画面片段和剧本
- 形成 `DTC Creative Reference Pack`
- 生成一张横屏 5 格 16:9 故事版图片
- 故事版具备真实拍摄感和动作连续性
- 输出英文视频 demo 脚本
- 脚本包含 Camera / Motion 和 Reference Logic
- 商品始终是主角
