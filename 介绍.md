# DTC 商品创意故事版 Skill 中文介绍

## 一、定位

`storyboard-dtc-product-creative` 是一个 SKU 级 DTC 商品创意故事版 skill。

它的目标不是单纯写广告文案，也不是只做一张好看的图，而是把一个真实商品证据转成可继续进入 StoreClaw、Amazon listing demo、Shopify PDP video 或 paid social 测试的故事版包。

核心路径：

```text
Product Truth Card
-> 合规参考采集
-> 广告参考拆解
-> DTC Creative Director
-> DTC Creative Reference Pack
-> Brand Film Look Director
-> Brand Film Look Pack
-> Storyboard Design Expert
-> 横向 5 格 16:9 故事版
-> 英文 15s/30s 视频脚本
-> 本地验证
```

## 二、适用输入

可以输入：

- Amazon 商品链接或 ASIN
- Shopify 商品链接
- TikTok Shop 商品链接
- 商品图
- 商品截图
- SKU brief
- 商品短描述
- 用户提供的参考广告、截图、品牌素材或脚本

输入越接近真实商品证据，输出越稳定。只有短描述时，不能把结果当成最终商品故事版。

## 三、默认输出

最终用户可见输出只保留三段：

```markdown
## 1. 产品图

## 2. 故事版

## 3. 15 秒或者 30 秒视频脚本文字
```

`## 2. 故事版` 嵌入或链接一张 5 格 contact sheet，每格都是 16:9 横屏 video-ready keyframe。

脚本文字默认 15 秒；用户要求 30 秒时输出 30 秒。脚本内容使用英文，表头可以保留中文。

## 四、执行合同

这个 skill 运行时直接产出工作记录和交付物，不只输出建议。

每次执行必须：

- 建立 `Product Truth Card`
- 记录 `source_records.json`
- 拆解可用参考为 `ad_reference`
- 形成 `DTC Creative Reference Pack`
- 形成 `Brand Film Look Pack`
- 生成或组装 `contact_sheet.png` 和 `panels/*.png`
- 保存 `run.json`
- 运行 `scripts/validate_storyboard.py`
- 如果当前环境缺少抓取、生图、文件写入或图片嵌入能力，在 `run.json` 记录 blocked artifact 和原因

## 五、三个内部角色

### DTC Creative Director

它负责出图前的创意选择，不负责最终出图。

固定选择顺序：

```text
1. creative_type
2. scene_ref
3. motion_chain
4. model_ref
5. camera_plan
6. brand_film_mode
7. script_pattern
8. picture_fragment_logic
9. end_frame
10. output DTC Creative Reference Pack
```

它的唯一交付物是 `DTC Creative Reference Pack`。

### Brand Film Look Director

它负责把 `brand_film_mode` 从“风格方向”翻译成可执行的拍摄规则，不负责重新选商品逻辑，也不改变商品事实。

它产出的 `Brand Film Look Pack` 说明：

- 这个品牌片方向具体是什么视觉温度和色彩关系
- 镜头应该怎么动，距离应该多近
- 光从哪里来，阴影和接触感怎么证明真实商品
- 材质、反光、纹理应该如何被看见
- 哪个画面承担 signature shot
- 收尾画面如何建立购买信心
- 哪些画面必须拒绝或重生成

它不能在最终输出里写“像某导演”或暴露导演名，只能输出品牌片语言。

### Storyboard Design Expert

它只使用 `DTC Creative Reference Pack` 和 `Brand Film Look Pack`，不重新选择创意方向。

它负责：

- 生成横屏 5 格 16:9 故事版图片
- 写英文 15 秒或 30 秒视频脚本
- 保持脚本、镜头、运动和故事版一致

## 七、结构化创意库

这一部分是 skill 的可选项菜单。每个条目都带 `description`，用中文直接说明这个选项干什么；Reference Pack 选择时也必须把这些中文说明带过去。

### creative_types

位置：

```text
references/dtc-creative-pattern-library.md
```

creative_type 决定这条广告用什么购买逻辑推进：解决问题、消除疑虑、展示功能、解释机制、做对比、证明尺寸适配、进入日常、展示定制或建立信任。

| ID | 中文说明 |
|---|---|
| `problem-solution` | 用商品使用过程把一个具体问题转成解决状态。 |
| `objection-proof` | 用细节、尺寸、材质或操作消除购买疑虑。 |
| `feature-to-benefit` | 把一个可见功能翻译成用户能理解的好处。 |
| `mechanism-reveal` | 展示结构、材质或工作机制，让用户知道价值来源。 |
| `comparison` | 用真实或中性对比帮助用户做选择。 |
| `scale-proof` | 用尺寸、比例、贴合或实物参照证明商品是否适配。 |
| `routine-integration` | 展示商品如何自然进入日常流程。 |
| `customization-choice` | 展示用户如何选择、配置或定制商品。 |
| `trust-proof` | 用可见细节和处理动作建立质量信任。 |
| `sensory-proof` | 用微距、触摸或光线呈现材质和质感。 |
| `product-macro` | 用商品本身的细节、结构和包装完成吸引与证明。 |
| `ugc-demo` | 用真人或手部演示解释商品使用方法。 |
| `founder-demo` | 用品牌或创始人理由连接产品设计细节和信任。 |

每个 `creative_type` 都包含：

```text
id
description
label
best_for
buyer_tension
required_product_evidence
visual_structure
hook_logic
proof_logic
CTA_logic
risk
when_not_to_use
```

买家问题到 `creative_type` 的选择路由：

| 买家或商品情况 | 首选 creative_type | 备选 creative_type | 什么时候用 |
|---|---|---|---|
| 商品需要解释怎么用 | `ugc-demo` | `routine-integration` | 使用方法、操作顺序、上手方式或真实使用语境需要被看见 |
| 买家担心质量 | `trust-proof` | `mechanism-reveal` | 材质、结构、包装、耐用感、工艺或可信细节是购买阻力 |
| 买家担心尺寸适配 | `comparison` | `scale-proof` | 尺寸、容量、比例、贴合、佩戴、安装或收纳适配需要被证明 |
| 商品有定制或选项 | `customization-choice` | 无 | 颜色、尺寸、套装、模块、配置、上传图案或个性化入口必须被看见 |
| 商品材质、细节强 | `sensory-proof` | `product-macro` | 质地、纹理、光泽、触感、结构、包装或工艺是主要吸引点 |
| 商品使用步骤重要 | `feature-to-benefit` | `mechanism-reveal` | 用户需要理解功能入口、步骤顺序、机制细节和最终好处 |

## 九、Brand Film Look Pack

`Brand Film Look Pack` 是 Brand Film Look Director 到 Storyboard Expert 的品牌片视觉交接物。

它解决的问题是：不要只写“cinematic / premium / luxury / high-end”，而是把这些词拆成可以被图片生成和视频生成执行的规则。

核心字段：

```yaml
brand_film_look_pack:
  brand_film_mode:
  brand_world_lock:
  cinematic_grammar:
  shot_function_map:
  material_and_light_rules:
  pacing_rules:
  signature_shot:
  end_frame_rule:
  failure_rules:
```

字段含义：

| 字段 | 中文说明 |
|---|---|
| `brand_film_mode` | 选择的品牌片视觉模式，只用品牌片语言，不暴露导演名。 |
| `brand_world_lock` | 锁定同一个拍摄世界，包括色温、光源、背景密度、人物存在方式和道具策略。 |
| `cinematic_grammar` | 把镜头、焦距感、机位、运动、光线、阴影、色彩和剪辑节奏写成可执行规则。 |
| `shot_function_map` | 把每个 panel 的视觉任务映射到 `picture_fragment_logic`，不固定某个 panel 必须承担某个功能。 |
| `material_and_light_rules` | 规定主光、补光、接触阴影、反光、纹理和商品边缘高光，防止 AI 塑料感。 |
| `pacing_rules` | 定义整体节奏，帮助 5 格故事版像同一条视频的关键帧。 |
| `signature_shot` | 选择最能代表这条商品创意的关键画面，但它必须服务商品证明。 |
| `end_frame_rule` | 规定购买信心收尾画面的商品位置、logo/label、proof memory、CTA 意图、镜头和光线状态。 |
| `failure_rules` | 写清楚哪些画面必须拒绝，比如商品变形、导演名外露、只有氛围没有证明。 |

没有这个 pack，不进入 Storyboard Design Expert。

## 十、横屏 5 格故事版规则

横屏故事版仍然输出 5 格，但不固定每一格的剧情角色。

每格的 `content_task`、镜头、动作和视觉片段来自 `DTC Creative Reference Pack` 里的 `picture_fragment_logic`，并由 `Brand Film Look Pack` 翻译成具体镜头、光线、材质和节奏规则。

默认节奏建议，不是硬性模板：

```text
quiet hook
brand world / buyer context
product proof moment
motion or use completion
restrained hero frame
```

如果商品事实、`creative_type` 或可用 proof 需要不同顺序，以 `picture_fragment_logic` 为准，并在里面说明原因。

购买信心收尾画面必须继承 `end_frame`，但不写死某个 panel 编号：

```yaml
end_frame:
  product_position:
  logo_or_label_visibility:
  proof_memory:
  CTA_intent:
  camera_state:
  lighting_state:
```

每格都要：

- 保持同一个商品事实
- 保持同一拍摄世界
- 对应 motion 和 camera
- 能继续作为视频关键帧
- 避免 fake UI、fake claims、fake reviews

## 十二、验证

运行：

```bash
python3 scripts/validate_storyboard.py /path/to/output_dir
```

验证内容：

- `run.json` 存在且是有效 JSON
- `source_records.json` 存在且是有效 JSON
- `run.json` 包含 `source_url` 和 `locked_terms`
- `brand_film_mode` 存在，并且不把导演名字暴露给最终用户
- `brand_film_look_pack` 存在，并且包含品牌世界锁、电影语法、shot function map、材质光线规则、signature shot 和 end frame rule
- `end_frame` 包含商品位置、logo/label 可见性、proof memory、CTA 意图、镜头状态、光线状态
- contact sheet 存在且可读取
- 5 张 panel 都是 16:9
- `camera_plan` 五个槽位存在

注意：validator 检查文件和结构，不替代人工或多模态视觉判断。

## 十四、完成标准

一次完整运行必须得到：

- Product Truth Card 已建立
- 商品事实优先于创意参考
- 参考来源或 fallback 已记录
- 已形成 `DTC Creative Reference Pack`
- 已形成 `Brand Film Look Pack`
- 每个 selected creative/scene/motion/model/camera 都带中文 `description`
- Storyboard Design Expert 只使用 Reference Pack 和 Brand Film Look Pack
- 生成横向 5 格 16:9 故事版
- 英文脚本包含 `Camera / Motion` 和 `Reference Logic`
- 本地保存 `run.json` 和 `source_records.json`
- validator 通过
